```bash
ssh -i ~/.ssh/zdm.pem azureuser@10.1.0.8
sudo su - zdmuser
```

### Step 2 — Clone repo and navigate to Step3

```bash
# If repo not yet cloned:
cd ~
git clone <your-fork-url>
cd ~/GHCP-ODAA-PromptMigration/Artifacts/Phase10-Migration/ZDM/ORADB/Step3
```

```bash
bash zdm_commands_ORADB.sh init
# Then edit ~/zdm_oci_env.sh and populate the OCI OCID values
```

```bash

# If TDE is enabled:
# read -sp "TDE wallet password: " SOURCE_TDE_WALLET_PASSWORD; echo; export SOURCE_TDE_WALLET_PASSWORD
```

### Step 5 — Source OCI environment variables

```bash
source ~/zdm_oci_env.sh
```

```bash

```

### Step 7 — Run evaluation (dry run — no changes made)

```bash

# Review evaluation output; fix any reported issues before proceeding
```

## Important Notes

### Security
- ⚠️ **Never** commit passwords to this repository.
- Password files under `~/creds/` must be created at runtime from environment variables and deleted immediately after the migration completes.
- The `~/.azure/zdm_blob_creds` file on the ZDM server contains the Azure Blob access key — ensure it has permissions `600` and is owned by `zdmuser`.
- The RSP file (`zdm_migrate_ORADB.rsp`) uses `${VARIABLE}` placeholders for secrets. Run `envsubst` before use, or use the `zdm_commands_ORADB.sh` script which handles substitution automatically.

### Rollback
- Before switchover: ZDM can be aborted (`bash zdm_commands_ORADB.sh abort <JOB_ID>`); source database is unaffected.
- After switchover begins: Switchover is a controlled operation. If it fails midway, the source database (now a standby) can be re-promoted. Coordinate with Oracle ODAA support.
- See `ZDM-Migration-Runbook-ORADB.md` Phase 7 for detailed rollback procedures.

### Network
- ICMP (ping) to target ODAA (10.0.1.160) is blocked — this is expected. SSH (port 22) and Oracle listener (port 1521) are confirmed OPEN.
- Bandwidth: Source DB is 2.15 GB @ ~0.17 GB/hr redo generation rate. Initial backup transfer will complete in minutes. Network is not a bottleneck.
