# Artifacts Directory

This directory is intentionally empty in the repository.

## Purpose

When you run the prompts from the `prompts/` directory, artifacts will be generated and saved here automatically.

## Directory Structure (After Running Prompts)

After running the ZDM migration prompts, you will see a structure like this:

> **IMPORTANT:** All paths are relative to the repository root. Do NOT add subdirectories
> for the migration tool name (e.g. `ZDM/`) or project/database name. Artifacts are saved
> directly under `Artifacts/Phase10-Migration/Step<N>/`.

```
Artifacts/
└── Phase10-Migration/
    ├── Step1/                              # Generated by Step 1 (local session)
    │   └── remote-ssh-setup-report.md      # Remote-SSH setup report
    ├── Step2/                              # Generated by Step 2
    │   ├── ssh-config.md                   # SSH connectivity config (written after interactive collection)
    │   ├── Scripts/
    │   │   └── zdm_test_ssh_connectivity.sh
    │   └── Validation/
    │       ├── ssh-connectivity-report-<timestamp>.md
    │       └── ssh-connectivity-report-<timestamp>.json
    ├── Step3/                              # Generated by Step 3
    │   ├── db-config.md                    # Database and ZDM config (written after interactive collection)
    │   ├── Scripts/                        # Discovery scripts
    │   │   ├── zdm_source_discovery.sh
    │   │   ├── zdm_target_discovery.sh
    │   │   ├── zdm_server_discovery.sh
    │   │   ├── zdm_orchestrate_discovery.sh
    │   │   └── README.md
    │   └── Discovery/                      # Outputs after script execution
    │       ├── source/
    │       ├── target/
    │       └── server/
    ├── Step4/                              # Generated by Step 4
    │   ├── Discovery-Summary.md
    │   └── Migration-Decisions.md
    ├── Step5/                              # Generated by Step 5
    │   ├── Issue-Resolution-Log.md
    │   └── Verification-Results.md
    └── Step6/                              # Generated by Step 6
        ├── zdm_migrate.rsp
        ├── zdm_commands.sh
        └── ZDM-Migration-Runbook.md
```

## Getting Started

1. Clone this repository
2. In a **local** VS Code session, run `@Phase10-Step1-Setup-Remote-SSH` — it will check the Remote-SSH extension and configure your SSH connection, writing `Step1/remote-ssh-setup-report.md`
3. Connect VS Code to your ZDM jumpbox via the **Remote-SSH** extension (as `zdmuser`)
4. Run `@Phase10-Step2-Configure-SSH-Connectivity` — it will collect SSH settings interactively and write `Step2/ssh-config.md`
5. Run `@Phase10-Step3-Generate-Discovery-Scripts` — it will collect database settings interactively and write `Step3/db-config.md`
6. Pre-populate either config file to skip interactive collection on subsequent runs

## Note

Do not commit generated artifacts back to this repository. Add your project-specific artifact directories to `.gitignore` if needed.
